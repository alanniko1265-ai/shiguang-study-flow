# 拾光（Study Flow）开发文档

## 1. 产品定位

“拾光”是一款本地优先的学习记录与激励工具。它帮助使用者：

- 准确记录一次学习活动的投入时长；
- 从时间、分类、趋势和目标完成度等维度复盘；
- 用连续学习、每日目标和阶段反馈形成正向激励；
- 在不依赖账号和云服务的前提下长期保存、导入和导出数据；
- 后续平滑迁移到 Android，而不重写核心业务逻辑。

首版坚持“记录动作足够轻，反馈足够直观”。不加入社交、排行榜或复杂积分系统。

## 2. 首版功能范围

### 2.1 专注计时

- 正计时：开始、暂停、继续、完成；
- 选择学习分类、填写当前任务；
- 计时状态刷新页面后仍可恢复；
- 完成后自动生成一条学习记录；
- 小于 10 秒的误触记录不保存。

### 2.2 数据记录

- 每条记录包含分类、任务、开始时间、结束时间和实际时长；
- 支持手动补记；
- 支持删除错误记录；
- 默认分类：专业学习、语言学习、阅读、项目实践；
- 支持自定义新增分类和颜色。

### 2.3 数据统计

- 今日 / 近 7 天 / 近 30 天三个时间范围；
- 总投入、日均投入、学习次数、最长单次；
- 分类投入占比（环形图）；
- 每日投入趋势（柱状图）；
- 近 12 周学习热力图；
- 连续学习天数和每日目标完成度。

### 2.4 本地数据管理

- 数据存储在浏览器 `localStorage`，采用带版本号的数据结构；
- JSON 文件导出和导入；
- 无服务端、无账号、无网络请求；
- 首次使用提供少量演示数据，便于理解统计页；用户可在设置中清空。

## 3. 信息架构

- **今日**：当前计时、今日目标、分类快捷入口、最近记录；
- **统计**：关键指标、投入趋势、分类分布、学习热力图；
- **记录**：完整历史、手动补记、删除；
- **设置**：每日目标、分类管理、数据导入导出与清空。

## 4. 技术架构

### 4.1 技术选型

- React 19 + TypeScript：组件和业务逻辑可复用；
- Vite：本地开发和静态构建；
- Tauri 2：Windows 本地应用壳，并为 Android 构建预留官方路径；
- Lucide React：统一、轻量的线性图标；
- 原生 SVG：统计图表，无额外图表运行时负担；
- CSS 变量 + 响应式布局：桌面和移动端共享一套视觉系统。

### 4.2 分层

```text
UI 组件
  ↓
应用状态与统计选择器
  ↓
领域模型（Session / Category / Settings / ActiveTimer）
  ↓
StorageAdapter（首版为 LocalStorageAdapter）
```

UI 不直接操作 `localStorage`。持久化集中在 `src/lib/storage.ts`，后续 Android 可替换为 Tauri Store 或 SQLite 适配器。

### 4.3 数据模型

```ts
type StudySession = {
  id: string;
  categoryId: string;
  task: string;
  startedAt: string;
  endedAt: string;
  durationSeconds: number;
};

type Category = {
  id: string;
  name: string;
  color: string;
};

type ActiveTimer = {
  categoryId: string;
  task: string;
  startedAt: string;
  accumulatedSeconds: number;
  runningSince: string | null;
};
```

存储根对象包含 `schemaVersion`，迁移函数负责将旧数据升级到当前版本。

## 5. Android 扩展方案

1. 保持领域类型、统计函数和 React 组件不依赖浏览器专属 API；
2. 将文件下载、文件选择等能力封装在平台接口中；
3. Android 阶段运行 `tauri android init` 生成原生工程；
4. 将 LocalStorageAdapter 替换为 Tauri Store 或 SQLite Adapter；
5. 使用 Tauri 通知插件加入专注结束通知；
6. 针对 Android 加入返回键、状态栏、安全区和后台计时验证；
7. 若需要多设备同步，再单独增加可选同步层，不改变本地优先原则。

## 6. 视觉规范

- 气质：安静、清醒、克制；
- 主色：深墨绿；强调色：暖绿色；背景：低饱和米白；
- 标题使用衬线字体增强仪式感，正文使用系统无衬线字体保证可读性；
- 卡片保持轻边框和极弱阴影；
- 动画只用于计时状态、进度和页面切换；
- 移动端改为底部导航，所有点击目标不小于 44px。

## 7. 验收标准

- 开始、暂停、继续和完成计时均工作正常；
- 页面刷新后，运行中的计时不会丢失或停止累计；
- 新记录立即反映到今日、统计和历史列表；
- 三种时间范围的统计口径一致；
- 导出的 JSON 可重新导入并恢复数据；
- 390px 宽度下无横向滚动，主要操作可触达；
- `npm run build` 与 TypeScript 检查通过；
- Tauri 配置可被识别，后续可初始化 Android 工程。
